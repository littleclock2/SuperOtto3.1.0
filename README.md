# SuperOtto v3.1.0 - 优化版

## 版本说明

本版本是基于 SuperOtto 3.0.1 的优化升级版，主要改进包括：

### 1. 自动创建依赖文件夹
- **无需手动创建 `picturetest` 文件夹**
- 程序启动时会自动检测并创建 `picturetest` 和 `music` 文件夹
- 如果图片资源不存在，会自动切换到纯色背景模式

### 2. Bug 修复
- 修复了坐标验证的边界检查问题
- 修复了障碍物生成时的无限循环风险
- 修复了音乐播放器的空行读取问题
- 改进了 A* 算法的性能优化

### 3. UI 优化
- 优化了颜色方案，使用更现代的配色
- 浅蓝色背景 (RGB 240, 248, 255)
- 金色奖杯 (RGB 255, 215, 0)
- 橙色机器人1 (RGB 255, 140, 0)
- 紫色机器人2 (RGB 148, 0, 211)
- 改进了网格线显示效果

### 4. 代码质量改进
- 添加了坐标验证函数
- 改进了错误处理
- 优化了范围高亮计算（使用曼哈顿距离）
- 添加了最大尝试次数防止死循环

## 使用方法

### 直接使用（推荐）
1. 直接编译运行即可
2. 程序会自动创建所需的文件夹结构

### 图片资源（可选）
如果需要使用自定义图片，请将以下图片放入 `picturetest` 文件夹：
- `bk.JPG` 或 `bk.jpg` - 背景图片
- `Wall2.png` - 障碍物图片
- `Cup.png` - 奖杯图片
- `Wall3.png` - 终点图片
- `Wall4.png` - 起点图片
- `1.png` - 机器人1图片
- `2.png` - 机器人2图片
- `path.png` - 路径图片
- `range.png` - 范围高亮图片

### 音乐资源（可选）
将音乐文件（.mp3, .wma, .wav）放入 `music` 文件夹

## 编译说明

使用 Visual Studio 2022 打开 `SuperOtto.sln` 解决方案文件进行编译。

## 主要改进点

1. **依赖管理**：不再需要手动创建文件夹和复制资源文件
2. **错误处理**：增加了更多的输入验证和错误提示
3. **性能优化**：改进了范围高亮计算算法
4. **UI美化**：使用更现代的颜色方案
5. **代码健壮性**：增加了边界检查和防死循环机制
